Custom Macro:

  • Expanded Variable Ranges - A wider range of variables allows for more diverse controls:
    Null #0 Local #1 - #33 Common #100 - #999 System > #1000
  • Variable Aliasing - You can simulate control-specific system variables by aliasing Fanuc Macro B system variables to your custom identifiers using the Custom Macro feature (settings section):
    @12000 = #5012
    and then use e.g.:
    X#12000 (#12000 will behave as #5012)
  • Custom Cycles – If the simulator does not support the cycles used by your control or you want to override existing cycle, you can create your own using Fanuc Macro B/Centroid using the Custom Macro feature (settings section):
    #10000 = 83 (one-time call) #10000 = -83 (the modal call remains active until it is canceled by G67) O10000 ... M99
    When G83 is used, then O10000 will be called.

    Good to know

    Variables that can be used for G-code Macro Call: #10000 - #10010
    Variables that can be used for M-code Macro Call: #20000 - #20010

    Important

    You can only use one custom macro call (G-code or M-code macro call) per line. The key point is that while multiple standard G or M codes can appear in the same line, only one custom macro call (whether G-code or M-code) is typically allowed per line to avoid command conflicts and ensure proper execution.
  • Syntax – Mixed syntax is also supported, providing flexibility across different controls e.g.:
    IF #10 == #11 THEN #10=0 IF [#10 EQ #11] THEN #10=0
    or mix syntax of both above e.g.:
    IF [#10 == #11] THEN #10=0
  • Variable Initialization List If your macro requires certain variables to start with predefined values, you can initialize them easily:
    #100 = 10.0 #100 = 10.0 * 50 #100 = 10.0 * SIN[50]
  • Human-Readable Variable Names – In long macros, it's easy to lose track of what each variable represents. Descriptive aliases for common variables greatly improve code readability:
    #DIAM = 10 #HOLES_QUANTITY = 6 #ANGLE = 30

    Important

    This syntax cannot be used in Custom Cycles.
  • Flexible Variable Usage in Addresses The system supports multiple syntaxes when using variables in axis commands:
    X#10*6 Y#11*2 X#10 * 6 Y#11 * 2 X[#10 * 6] Y[#11 * 2]
